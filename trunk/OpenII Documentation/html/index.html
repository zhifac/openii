Welcome to OpenII

OpenII is a collaborative effort spearheaded by The MITRE Corporation to create a suite of open-source tools for information integration. The project is leveraging the latest developments in research on information integration to create a platform on which integration applications can be built and further research can be conducted. Our goal is to create tools that can be applied in a variety of architectural contexts and can easily be tailored to the needs of particular domains. OpenII tools include, among others, wrappers for common data sources, tools for creating matches and mappings between disparate schemas, a tool for searching collections of schemas and extending schemas, and run-time tools for processing queries over heterogeneous data sources.

Work Processes

Using The Tools
  • Affinity

    Affinity is used for finding associations between members of a generic group of data sources. Through this tool, users can see which data sources are closely associated, inspect the reason for these associations, and then perform actions on these closely related data sources.

  • Proximity

    Proximity is used to examine the exact alignment of a single data source with other data sources in a selected group. This view allows one to see a ranking of the most closely aligned data sources as well as detailed information on why these alignments took place.

  • Harmony

    Harmony is used to generate mappings between pairings of data sources. This tool takes a semi-automated approach where algorithms attempt first guess estimates of associated data fields. Then, users can examine these linkages to verify and annotate these linkages.

  • Unity

    Unity is used to create a collective vocabulary for a group of schemas by examining the mappings generated by tools such as Harmony. This vocabulary can then be used to help refine future matching inside of the domain or to help construct a messaging standard through which the schemas can communicate.

  • RMap

    RMap is used to generate transformation code between a pair of relational schemas. The resultant SQL code can be used to take data from the source schema and transform it into a format which can be accepted by the target schema.

Developers Zone
  • The M3 Model

    Underlying the entire OpenII repository is the M3 model. This model represents both schemas as well as the mappings between these schemas. This section discusses OpenII's data model, how to access this data model, and how this will help in the construction of the various OpenII plugin components.

  • Constructing Importers/Exporters

    This discusses how to construct new importers and exporters for use in OpenII. This section includes demonstrations on how to build importers and exporters for schemas, mapping, as well as projects.

  • Defining Schema Models

    In OpenII, the element hierarchy within a data source can be displayed in various ways. This section talks about the various schema models which are currently available as well as how to construct new schema models to represent schemas in a custom way.

  • Building Matchers/Mergers

    Underlying the Harmony program is a series of matchers and mergers which automatically generate mappings between data sources. Learn how to construct new matchers as well as the mergers which generates a concensus between these matchers.

  • Building Viewers/Editors

    OpenII runs in the Eclipse framework which provides a flexible framework for adding new viewers and editors to help interpret and work with the data in the selected repository. This section demonstrates how a new module can be constructed and the basics on how to connect it with all of the information in the data repository.
